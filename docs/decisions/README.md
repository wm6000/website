# Architecture Decision Records

This directory holds lightweight ADRs (Architecture Decision Records) — short documents capturing a significant technical decision, the alternatives considered, and why the chosen option won. Not every decision needs one; use this for choices that would otherwise get re-litigated or re-discovered by digging through commit history.

## Format

One file per decision, numbered sequentially: `NNNN-short-title.md`.

```markdown
# NNNN. Title

Status: Proposed | Accepted | Superseded by NNNN

## Context

What problem or question forced this decision?

## Decision

What was decided.

## Consequences

What this makes easier or harder going forward; any tradeoffs accepted.
```
